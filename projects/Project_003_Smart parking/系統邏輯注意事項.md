# 系統邏輯注意事項（Business Rules）

請嚴格遵守以下停車場管理規則：

1. **同一車牌不可重複進場**

   * 若車牌已有「未出場」紀錄，禁止再次建立進場紀錄。
   * 系統需提示：

   ```
   此車輛目前已在停車場內，無法重複進場。
   ```

2. **找不到進場紀錄不可出場**

   * 若輸入車牌查無未出場紀錄，禁止出場。
   * 系統需提示：

   ```
   查無有效進場紀錄。
   ```

3. **黑名單車輛處理**

   * 黑名單車輛進場時禁止放行。
   * 黑名單車輛出場時需顯示警告訊息。
   * 後台需記錄異常事件。

4. **月租車規則**

   * 月租車在有效期限內：

     * 停車費 = 0
     * 可直接出場
   * 若已過期：

     * 自動改為臨停車計費。

5. **VIP 車輛規則**

   * VIP 車輛停車費固定為 0 元。
   * 可直接出場。

6. **臨停車計費規則**
   預設：

   ```
   免費時間：30分鐘
   每小時：40元
   未滿1小時以1小時計
   每日最高收費：300元
   ```

7. **付款完成才能出場**

   * 臨停車必須先付款。
   * 未付款時禁止開啟出口柵欄。

8. **出場後不可重複付款**

   * 已出場紀錄禁止再次付款。
   * 若重複付款需提示：

   ```
   此停車紀錄已完成結帳。
   ```

9. **車位數量限制**

   * 剩餘車位不得小於 0。
   * 已使用車位不得超過總車位數。
   * 進場時：

   ```
   used_spaces += 1
   available_spaces -= 1
   ```

   * 出場時：

   ```
   used_spaces -= 1
   available_spaces += 1
   ```

10. **停車場已滿**

    * 當 available_spaces = 0 時：

    ```
    停車場已滿，暫停進場。
    ```

11. **付款成功流程**

    ```
    更新付款狀態 = 已付款
    建立付款紀錄
    更新停車紀錄狀態 = 已出場
    更新出場時間
    更新車位數
    ```

12. **異常事件紀錄**
    系統需記錄：

    * 重複進場
    * 黑名單進場
    * 非法出場
    * 重複付款
    * 設備異常
    * 車位數異常

    並保存：

    ```
    發生時間
    車牌號碼
    異常類型
    詳細描述
    操作者
    IP 位址
    ```
