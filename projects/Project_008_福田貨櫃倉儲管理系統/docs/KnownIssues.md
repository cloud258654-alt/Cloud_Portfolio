# 已知限制 — 福田貨櫃倉儲管理系統 v0.5.1

> 以下項目為目前已知但尚未修正的系統限制，不影響核心功能使用。

| 項目 | 影響範圍 | 嚴重度 | 目前處理方式 | 未來建議 |
|------|----------|--------|-------------|----------|
| Node 18 + Vite 7 版本警告 | 每次 `npm run build` 顯示黃色警告 | 🟡 低 | 保留，不影響 build | 升級 Node.js 至 20.19+ 或 22.12+ |
| 無後端伺服器 | 所有資料只能在單一瀏覽器使用 | 🔴 高 | 標示為「前端 Demo」 | v1.0 後可評估串接 REST API |
| 無資料庫 | 重設瀏覽器或清除快取後資料消失 | 🔴 高 | LocalStorage 暫存 | v1.0 後可接 PostgreSQL / Supabase |
| 無使用者登入權限 | 任何人打開網頁都能操作所有功能 | 🔴 高 | 標示為「管理員單一角色」 | v1.0 後補 JWT 或 OAuth |
| LocalStorage 不適合正式多人使用 | 多個瀏覽器/裝置資料不同步 | 🟠 中 | 匯出/匯入 JSON 手動同步 | 後端 + 資料庫 |
| 合約條款內容為展示用 | 法律效力不足，條文可能不完整 | 🟠 中 | 文件註明「需法務確認」 | 委由正式法務審閱合約模板 |
| AI 助理為 keyword-based | 無法理解複雜語意或上下文 | 🟠 中 | 文件說明為「規則引擎」 | 可串接 OpenAI API 或本地 LLM |
| 退租案件無獨立 state | 退租數量無法精確追蹤 | 🟡 低 | 依合約到期日推導 | 建立 checkout 獨立 data model |
| App.jsx 單檔架構（~2000 行） | 維護難度隨功能增加上升 | 🟠 中 | v0.5.1 保留現況 | v1.0 拆分為 component/page/hook |
| `<Table>` 組件使用 `key={index}` | 排序/過濾後 DOM 狀態可能錯位 | 🟡 低 | 現無刪除/拖曳場景，暫不影響 | 改用 `item.id` 作為 key |
| 手機版表格需手動橫向滑動 | 行動裝置體驗較差 | 🟡 低 | `.table-wrap` 設 `overflow-x: auto` | 未來可做 responsive 列隱藏 |
| 部分文字使用 hardcoded demo 值 | 如「管理員」「現場人員」等 | 🟡 低 | 保留 | 未來可串接真實人員資料 |
| 無單元測試 | 每次手動測試耗時 | 🟡 低 | 目前靠人工 QA | v1.0 補 Jest + React Testing Library |
| 無 CI/CD pipeline | 無法自動化部署 | 🟡 低 | 手動 `npm run build` | 可接 GitHub Actions / Vercel |
