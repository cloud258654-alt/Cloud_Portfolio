# v0.5.1 Release Notes

## Summary

QA fix release addressing 5 of 7 known issues from the v0.5 QA report.  
No new features. No npm packages added. No backend. No database.

---

## Fixes

### 1. Unified Payment Calculation

Replaced all remaining `item.paidAmount` / `p.paidAmount` references across the codebase with `getPaidAmount(item)`, which correctly handles `partialPayments` arrays.

**Affected components:**
- ContainerDetail
- QrScanner
- CustomerManagement (customer center + rental summary)
- CustomerPortal
- Reports
- Dashboard (unpaid list)

**Rule:**
```
If payment.partialPayments exists → sum partialPayments amounts
Else → use payment.paidAmount
```

### 2. Import JSON Missing Field Warning

`handleImportJSON()` now detects missing fields (`reservations`, `payments`, `electricity`, `maintenance`, `notifications`) and warns the user:
> 資料匯入成功！注意：以下欄位未提供，已使用預設資料：帳單、電費。

If the JSON is completely malformed:
> 資料格式錯誤，無法解析 JSON。

### 3. AI Checkout Query Now Dynamic

Previously returned hardcoded demo data. Now:
- If occupied containers with contracts ending within 90 days exist → lists them
- If none → returns: "目前尚未建立正式退租案件資料來源..."

### 4. Accessibility: focus-visible Styles

Added to `src/styles.css`:
```css
button:focus-visible, input:focus-visible, select:focus-visible,
textarea:focus-visible, a:focus-visible {
  outline: 2px solid #f6c85f;
  outline-offset: 2px;
}
```
Keyboard navigation now has a clear gold outline; mouse clicks unaffected.

### 5. `.dashboard-grid.refined` CSS Defined

Added:
```css
.dashboard-grid.refined {
  grid-template-columns: minmax(0, 1fr) minmax(320px, 0.65fr);
}
```
