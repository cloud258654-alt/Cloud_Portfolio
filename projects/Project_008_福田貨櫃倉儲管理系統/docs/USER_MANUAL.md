## 1. 系統簡介

福田貨櫃倉儲管理系統是一套**貨櫃出租管理工具**。你可以在這裡：

- 看所有貨櫃的狀態（空的、租出去的、維修中的）
- 管理客戶資料與租金帳單
- 追蹤預約排隊與訂金
- 簽署合約並查看完整條款
- 試算退租費用並結案
- 用 AI 助理快速查詢營運狀況

所有資料會**自動儲存在你的瀏覽器**，關掉網頁重開不會消失。

## 3. Dashboard 首頁說明

打開系統後看到的第一頁就是 Dashboard。

### 3.1 KPI 數字卡

最上方 12 張卡片，顯示最重要的營運數字：

| 卡片 | 意思 |
|------|------|
| 本期應收租金 | 所有租金帳單的總金額 |
| 已收金額 | 客戶已經付的租金 |
| 未收金額 | 還沒收到的錢 |
| 逾期金額 | 已經過期還沒付的錢 |
| 押金總額 | 所有已收的押金 |
| 預約訂金 | 預約時收的訂金 |
| 空櫃數 | 目前可以出租的貨櫃 |
| 使用中 | 正在出租的貨櫃 |
| 維修中 | 正在維修的貨櫃 |
| 即將到期 | 30 天內租約到期的貨櫃 |
| 退租中 | 正在辦理退租的案件 |
| 逾期客戶 | 有帳單過期未繳的客戶人數 |

顏色意義：
- 🟢 綠色 = 正常
- 🔴 紅色 = 需要處理
- 🟡 黃色 = 注意
- 🔵 藍色 = 資訊

### 3.2 營運風險提醒

列出目前需要注意的事項，例如：
- 「李小姐 / B03 / 租金 $84,000 / 逾期 21 天」→ 紅色標籤
- 「黃先生 / 預約訂金 $4,000 待收」→ 黃色標籤

### 3.3 今日待辦

系統自動整理的工作清單，每項都有優先級：
- 🔴 **高優先**：逾期催收
- 🟡 **中優先**：續約聯絡、訂金跟進、維修檢查
- 🔵 **低優先**：候補通知

**點擊任何一項會跳到對應的功能頁面。**

### 3.4 資料管理

在 Dashboard 下半部，包含：
| 按鈕 | 功能 |
|------|------|
| Export JSON | 下載完整資料備份 |
| Import JSON | 匯入之前備份的資料 |
| CSV: Customers | 匯出客戶清單 Excel |
| CSV: Contracts | 匯出合約清單 Excel |
| CSV: Payments | 匯出帳單清單 Excel |
| Reset Demo Data | 清除所有資料，恢復初始示範資料 |

## 4. 貨櫃管理

### 查看貨櫃

左側選單點「貨櫃管理」，可以看到：
- 上方：貨櫃地圖（每個櫃子用彩色方塊顯示）
- 中間：貨櫃列表（可按規格、狀態、區域篩選）
- 右側：選取貨櫃的詳細資料

### 新增貨櫃

1. 找到右下「新增貨櫃」表單
2. 填入貨櫃編號（例如 C05）
3. 選規格（10 呎 / 20 呎）
4. 填入區域（A / B / C）
5. 選狀態（通常是「空櫃」）
6. 點「新增貨櫃」

> ⚠️ 編號不能重複，否則不會新增。

## 5. 客戶管理

### 查看客戶

左側選單點「客戶管理」，上方可選擇客戶，下方顯示該客戶的：
- 基本資料（姓名、電話、身分證、發票抬頭、公司名稱等）
- 目前租用的貨櫃
- 預約紀錄
- 合約
- 帳單與未收款

### 新增客戶

1. 找到右側「新增客戶」表單
2. 必填：姓名、電話
3. 建議填寫：身分證號、發票抬頭、公司名稱、聯絡人
4. 選擇付款偏好（匯款/現金/支票/信用卡）
5. 選擇通知偏好（LINE/電話/Email）
6. 點「新增客戶」

## 6. 預約管理

### 查看預約

左側選單點「預約管理」，上方是預約狀態看板：

| 欄位 | 意思 |
|------|------|
| 等待中 | 客戶在排隊等空櫃 |
| 保留中 | 已通知客戶，等待確認 |
| 已確認 | 客戶已確認要租 |
| 已轉租 | 已轉為正式合約 |
| 已失效 | 超過保留期限 |
| 已取消 | 客戶取消預約 |

### 通知客戶保留

1. 在「等待中」的預約旁，從下拉選單選擇一個空櫃
2. 點「通知保留」
3. 系統會模擬發送 LINE 通知給客戶
4. 預約狀態改為「保留中」

### 訂金狀態

- 🟢 綠色「已收 $6,000」→ 客戶已付訂金
- 🔴 紅色「待收訂金」→ 還沒收，要催收

## 7. 電子合約

### 查看合約

左側選單點「電子合約」，可以看到合約看板（草稿 / 待簽署 / 已簽署）和合約列表。

表格包含：合約編號、承租人、貨櫃、類型（首年/續約）、租金、押金、收款狀態、簽署時間等。

### 查看合約條款

1. 右側「合約條款」面板，從下拉選單選擇一個合約
2. 系統會顯示該合約的完整條款：

| 條款區 | 內容 |
|--------|------|
| 承租人 | 姓名、電話、身分證、地址 |
| 貨櫃資料 | 編號、規格、區域 |
| 租期與租金 | 起迄日期、首年/續約、年租金 |
| 押金規則 | 押金金額與退還原則 |
| 電費規則 | 基本費 1,000/年 + 每度 9 元 |
| 中途解約 | 通知天數與退款規則 |
| 損壞扣款 | 損壞賠償規則 |
| 退款規則 | 押金退還流程 |
| 簽署與版本 | 版本號、簽署時間、狀態 |

3. 若客戶有預約訂金，會顯示綠色提示「此客戶已有預約訂金：NT$X,XXX」

### 模擬簽署

未簽署的合約，點「模擬簽署」按鈕即可模擬簽約流程。

## 8. 收費管理

### 查看帳單

左側選單點「收費管理」，可看到：
- 未收總額、逾期筆數
- 所有帳單列表（帳單號碼、客戶、貨櫃、金額、已繳、餘額等）

### 產生年度帳單

點「產生年度租金帳單」，系統會自動為所有出租中的貨櫃產生下一年的租金帳單。

### 標記已繳

當客戶付款後：
1. 在未繳帳單的那一列，選擇付款方式（匯款/現金/支票/信用卡/LINE Pay）
2. 點「標記已繳」
3. 系統會自動記錄付款日期、收款人、更新收據狀態

### 部分付款

如果客戶分次付款，帳單會顯示「2 筆 / $30,000」的黃色標記。滑鼠移上去可以看到每一筆的明細。

### 收據與發票

| Badge | 意思 |
|-------|------|
| 🟢 已開立 | 收據/發票已開 |
| 🟡 待開立 | 尚未開立 |
| 🔵 免開立 | 不需開立（如現金電費） |

## 9. 電費管理

左側選單點「電費管理」，顯示每個貨櫃的電表讀數：

| 欄位 | 說明 |
|------|------|
| 貨櫃 | 貨櫃編號 |
| 上期度數 | 上次抄表度數 |
| 本期度數 | 這次抄表度數 |
| 用量 | 本期 - 上期 = 用電度數 |
| 基本費 | 固定 1,000 元/年 |
| 用電費 | 度數 × 9 元 |
| 總金額 | 基本費 + 用電費 |

> 大量用電客戶會裝獨立電表，每度 9 元計算。

## 11. 退租管理

### 試算退租費用

1. 左側選單點「退租管理」
2. 從下拉選單選擇要退租的貨櫃
3. 設定退租日期
4. 視情況輸入：
   - 損壞扣款（櫃子有損壞需扣的錢）
   - 清潔費（清潔費用）
   - 違約金（提前解約的罰款）
5. 系統會自動計算：

| 試算項目 | 說明 |
|----------|------|
| 合約租期 | 合約的起迄日期 |
| 已用月份 | 到退租日已使用幾個月 |
| 剩餘月份 | 合約還剩幾個月 |
| 年度租金 | 一年租金多少 |
| 剩餘租金退費 | 沒用完的月份按比例退費 |
| 押金 | 當初收的押金 |
| 未繳費用 | 還沒付的帳單 |
| 待收電費 | 還沒付的電費 |
| 損壞扣款 | 你輸入的金額 |
| 清潔費 | 你輸入的金額 |
| 違約金 | 你輸入的金額 |
| 扣款合計 | 以上所有扣款加總 |
| **最終應退** | 要退給客戶多少錢 |
| **最終應補** | 客戶還要補多少錢 |

### 結案摘要

試算表下方的「退租結案摘要」列出 15 項明細，包含結案後三個資源的狀態：
- 貨櫃 → 變成空櫃 (available)
- 合約 → 關閉 (closed)
- 客戶 → 若無其他租約則結案 (closed)，否則仍活躍 (active)

### 完成退租

確認試算無誤後，點「完成退租」按鈕即可結案。

## 12. AI 助理

### 使用方式

1. 點右下角浮動 AI 按鈕，或左側選單點「AI 助理」
2. 輸入問題，或直接點選建議問題

### 可以問的問題

| 問題範例 | AI 會回答 |
|----------|-----------|
| 目前有多少空櫃？ | 列出所有空櫃的編號、規格、區域 |
| 誰還沒繳錢？ | 列出未繳客戶與金額明細 |
| 逾期金額多少？ | 逾期總額 + 優先聯絡名單 |
| 本期收入多少？ | 應收 vs 已收 vs 未收 |
| 哪些合約快到期？ | 60 天內到期合約列表 |
| 有哪些預約還沒收訂金？ | 待收訂金明細 |
| 哪些貨櫃維修中？ | 維修中貨櫃與狀態 |
| 今天要處理什麼？ | 編號待辦清單 |
| 營運風險有哪些？ | 分類風險統計 |
| 目前共有幾位客戶？ | 客戶總數 |
| 目前共有幾份合約？ | 合約總數 |
| 目前共有多少帳單？ | 帳單總數 |
| 目前 Storage 是否正常？ | 儲存狀態、最後儲存時間 |

## 13. 匯出與匯入資料

### 匯出備份

1. 回到 Dashboard
2. 點「Export JSON」→ 下載完整資料備份檔（檔名：`FutianStorage_YYYYMMDD.json`）
3. 點「CSV: Customers」等按鈕 → 匯出 Excel 可開的報表

### 匯入還原

1. 點「Import JSON」
2. 選擇之前下載的 `.json` 備份檔
3. 系統會檢查格式：
   - ✅ 格式正確 → 匯入成功
   - ⚠️ 缺少部分欄位 → 會提示缺少哪些，並用預設資料補齊
   - ❌ 格式錯誤 → 提示錯誤

### 重置資料

點「Reset Demo Data」會清除所有資料，恢復成系統初始的示範資料。

## 14. 常見問題

### Q: 關掉網頁後資料會不見嗎？
A: 不會。系統會自動儲存在瀏覽器的 LocalStorage，重開網頁資料還在。

### Q: 換一台電腦或換瀏覽器，資料會同步嗎？
A: 不會。目前是單機版本。需要在兩台電腦間同步的話，請使用「Export JSON」匯出後再「Import JSON」匯入到另一台。

### Q: 為什麼有些按鈕點了沒反應？
A: 可能是模擬功能（如 LINE 通知、PDF 產生），目前為展示用途，不會實際發送或產生檔案。

### Q: 金額計算正確嗎？
A: 費率完全依照設定的規則計算（首年/續約費率、押金 = 1 個月租金、電費基本費 1,000 + 每度 9 元）。部分付款會自動加總。

### Q: 手機可以用嗎？
A: 可以。畫面會自動調整為手機適合的大小。表格可能需要左右滑動。
