# Antigravity Stock AI Ver 2.0 盤中即時資訊規劃

更新日期：2026-06-26

## 規劃定位

Ver 2.0 的下一步不只是「顯示即時股價」，而是把產品升級成盤中投資工作台。使用者在交易時間打開系統時，應該能立刻回答三個問題：

1. 現在市場正在發生什麼？
2. 這些變化是否影響原本的 AI 分析結論？
3. 我現在需要觀察、加碼、減碼，還是等待確認？

因此本版本將新增「盤中即時行情」、「盤中訊息流」、「AI 即時解讀」與「警示行動」四層能力，讓 Antigravity Stock AI 從靜態報告工具，升級為可陪使用者盯盤的決策儀表板。

## 核心模組

### 1. Live Market Tape 盤中報價帶

放置於首頁搜尋區下方、Market Dashboard 上方，作為第一眼市場狀態。

顯示內容：

- 加權指數、櫃買指數、台指期、美元台幣匯率。
- 重要權值股：2330 台積電、2317 鴻海、2454 聯發科、2382 廣達。
- 每筆資料顯示現價、漲跌、漲跌幅、成交量簡寫與最後更新時間。
- 右側顯示資料狀態：模擬資料、延遲資料、即時資料、連線異常。

互動設計：

- 點擊個股可直接帶入搜尋欄並啟動分析。
- 漲跌使用台股慣例：紅色上漲、綠色下跌。
- 每 5 至 15 秒刷新一次，首版使用 mock 資料模擬盤中跳動。

### 2. Intraday Snapshot 個股盤中快照

放置於個股報告頁的總結區附近，讓使用者先看盤中狀態，再讀完整分析。

顯示內容：

- 現價、漲跌幅、今日開高低、成交量、成交值。
- 今日區間位置：目前價格位於今日低點至高點的百分位。
- 相對大盤強弱：強於大盤、同步大盤、弱於大盤。
- 量能狀態：正常、放量、爆量、量縮。
- AI 盤中結論：維持評等、升評觀察、降評觀察、等待確認。

經理人視角：

- 價格突破但量能不足，不直接升評，只列為觀察。
- 放量突破關鍵均線，標記為偏多確認。
- 跌破停損線或重要支撐，優先提示風險。

### 3. Real-time Message Center 即時訊息中心

放置於首頁 Market Dashboard 或 Global Toolkit 中，作為盤中資訊流。

訊息分類：

- 重大訊息：MOPS 公告、法說會、財報、董事會、處分資產、增資減資。
- 新聞快訊：公司新聞、產業新聞、政策消息、國際事件。
- 技術訊號：突破均線、跌破支撐、爆量、接近漲停跌停。
- 籌碼訊號：法人買賣超、分點異常、大單成交、主力進出。
- 系統訊息：資料延遲、API 異常、更新完成。

每則訊息欄位：

- 時間
- 股票代號與名稱
- 訊息類型
- 標題
- 摘要
- AI 判讀：偏多、偏空、中性、雜訊
- 影響層級：短線、波段、基本面、風險控管

### 4. Watchlist Live Monitor 自選股即時監控

延伸目前已有的最近搜尋與自選股功能。

新增內容：

- 自選股即時價格、漲跌幅、成交量狀態。
- 自選股事件數量：今日新聞數、警示數、技術訊號數。
- 自選股排序：漲幅、跌幅、成交值、訊息密度、AI 評分變化。
- 重要變動置頂，例如觸價、爆量、重大訊息。

首版資料儲存：

- 使用 `localStorage` 保存自選股與使用者設定。
- 不需要登入即可使用。
- 未來可升級為雲端帳號同步。

### 5. Alert Center 警示中心

將現有策略警示擴充成可設定的盤中提醒。

警示類型：

- 價格高於 / 低於指定價。
- 漲跌幅超過指定百分比。
- 成交量超過近 20 日均量倍數。
- 突破或跌破 MA5、MA20、MA60。
- 接近 AI 停損線或目標價。
- 出現重大訊息或高影響新聞。
- AI 評等改變。

MVP 階段：

- 先做前端警示規則與畫面呈現。
- 警示結果在頁面內顯示，不做手機推播。
- 使用者關閉瀏覽器後，由 `localStorage` 保存規則。

## AI 解讀邏輯

AI 不只負責摘要訊息，而是要判斷「這件事是否改變投資假設」。

### 訊息判讀框架

每則訊息進入系統後，根據四個維度評估：

1. 方向性：偏多、偏空、中性。
2. 時效性：盤中短線、數日波段、中長期基本面。
3. 可信度：官方公告、主流媒體、市場傳聞、技術訊號。
4. 影響程度：低、中、高、重大。

### 對報告評等的影響

AI 盤中結論分為四種：

- 維持原評等：訊息不影響原本投資邏輯。
- 升評觀察：出現偏多訊號，但仍需成交量或收盤確認。
- 降評觀察：出現偏空訊號，但尚未破壞主趨勢。
- 立即風險提示：跌破停損、重大利空、流動性異常。

## 使用者流程

### 首頁流程

1. 使用者進入首頁。
2. Live Market Tape 先顯示大盤與主要權值股狀態。
3. Market Dashboard 顯示盤中熱點、類股強弱與即時訊息。
4. 使用者點擊個股或輸入股票代號。
5. 系統啟動 AI 分析流程。

### 分析中流程

1. Multi-Agent Pipeline 照常顯示代理人辯論。
2. 辯論日誌加入盤中訊息，例如「技術代理人偵測到量能放大」。
3. 若盤中有重大訊息，代理人會在分析中標記風險或機會。

### 報告頁流程

1. 報告上方先顯示 Intraday Snapshot。
2. AI 總結補上盤中結論。
3. 技術面、籌碼面、基本面各自引用盤中資訊。
4. 使用者可加入自選股或建立警示。

## 技術實作策略

### 第一階段：Mock Realtime MVP

目標是先完成完整體驗，不被外部 API 卡住。

實作內容：

- 新增即時報價資料模型。
- 以 `setInterval` 模擬每 5 至 15 秒刷新。
- 新增盤中訊息 mock data。
- 新增自選股即時監控。
- 新增警示規則判斷。
- 將盤中資料注入既有 `generateMockReport` 報告。

資料來源狀態標示為「模擬盤中資料」。

### 第二階段：API Proxy

真實行情資料不建議由前端直接呼叫，應加入輕量後端或 Serverless Proxy。

建議方案：

- Cloudflare Workers
- Vercel Functions
- Netlify Functions
- GitHub Pages + 外部資料 Proxy

Proxy 負責：

- 串接 TWSE / TPEX 公開資料。
- 串接 MOPS 重大訊息。
- 串接新聞來源或付費行情 API。
- 快取資料，降低請求頻率。
- 統一輸出前端可用 JSON。

### 第三階段：真正即時化

若未來需要更接近專業交易系統，可升級：

- WebSocket 即時推送。
- 使用者雲端自選股。
- 跨裝置警示同步。
- Email / LINE Notify / Browser Push 通知。
- 盤中策略回測與觸發紀錄。

## MVP 驗收標準

完成後應符合：

1. 首頁可看到盤中報價帶，且數值會定時更新。
2. 即時訊息中心可顯示不同類型訊息與 AI 判讀。
3. 自選股可顯示即時價格與訊號數。
4. 個股報告頁可顯示盤中快照。
5. 使用者可建立至少一種價格警示。
6. 手機版不發生文字重疊或卡片破版。
7. 即使沒有真實 API，也能以 mock data 完整展示產品體驗。
